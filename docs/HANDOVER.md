# Этап 1: сайт + сбор заявок (без админки)

Эта конфигурация запускает **публичный сайт** и **сбор заявок** — без
профессиональной CMS (редактора контента). Заявки приходят на почту и видны на
отдельной странице по токену. Это «версия для передачи клиенту».

В чём отличие от того, что сейчас на `new.icemr.biz`: там **статическая витрина**
(GitHub Pages) — формы только показываются, но никуда не отправляются. Чтобы
заявки действительно собирались, нужен **сервер с Node.js + база данных**.

## Что нужно для запуска

1. **Хостинг с Node.js** (не статический): Railway, Render, Timeweb Cloud, VPS
   и т.п. Для основного сайта РУДН лучше российский хостинг.
2. **PostgreSQL** — управляемая база или на том же сервере.
3. **Домен + SSL** — домен есть, подключается к хостингу (SSL обычно
   выпускается автоматически).

## Переменные окружения (.env)

Скопируйте `.env.example` → `.env` и заполните:

| Переменная | Назначение |
|---|---|
| `DATABASE_URL` | строка подключения к PostgreSQL |
| `AUTH_SECRET` | длинная случайная строка (`openssl rand -base64 48`) |
| `NEXT_PUBLIC_SITE_URL` | публичный адрес сайта, напр. `https://new.icemr.biz` |
| `ENABLE_CMS` | **оставить пустым** — так админка `/admin` отключена (404) |
| `LEADS_TOKEN` | секретный токен для просмотра заявок (мин. 8 символов) |
| `LEADS_NOTIFY_EMAIL` | куда слать уведомления о новых заявках |
| `SMTP_HOST` / `SMTP_PORT` / `SMTP_USER` / `SMTP_PASS` / `SMTP_FROM` / `SMTP_SECURE` | доступ к почтовому SMTP (Яндекс, Mail.ru и т.п.) |

Если SMTP не заполнен — уведомления просто не отправляются, остальное работает.

## Развёртывание

```bash
npm install
cp .env.example .env          # заполнить значения
npx prisma db push            # создать таблицы
npm run db:seed               # наполнить контентом (программы, преподаватели…)
npm run build
npm start                     # или процесс-менеджер хостинга
```

## Как смотреть заявки

- По ссылке с токеном (можно добавить в закладки):
  `https://<домен>/leads/auth?token=ВАШ_LEADS_TOKEN`
- Либо открыть `https://<домен>/leads` и ввести токен один раз.

На странице — список откликов (новые сверху), контакты кликабельны, у каждой
заявки можно выставить статус (Новая / В работе / Связались / Закрыта / Спам).
Плюс письмо на `LEADS_NOTIFY_EMAIL` при каждой новой заявке.

## Передача исходников клиенту (опционально)

Если вы **разворачиваете сайт сами** и отдаёте клиенту только работающий адрес —
ничего дополнительно делать не нужно: при пустом `ENABLE_CMS` админки нет.

Если же отдаёте **исходный код**, уберите из него CMS, чтобы редактор нельзя было
включить флагом:

```bash
bash scripts/make-handover.sh ../rudn-handover
```

Скрипт создаёт чистую копию проекта без кода админки (`src/app/admin`,
`src/components/admin`, `src/app/api/auth`, `src/app/api/media`). Останется
публичный сайт + сбор заявок + `/leads`.

## Персональные данные (152-ФЗ)

Формы уже требуют согласие на обработку ПДн, а страницы политики
конфиденциальности / согласия встроены. Для основного сайта РУДН данные заявок
желательно хранить на серверах в РФ.
